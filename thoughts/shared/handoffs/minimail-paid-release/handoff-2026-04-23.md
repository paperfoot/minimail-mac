## Key decisions made in this batch (not obvious from code)

1. **Resend rewrites Message-ID (deal-breaker for clean threading).** AmazonSES (Resend's underlying) replaces our locally-generated `Message-ID` header on outbound mail. Their API doesn't return the delivered ID either. So any header-chain-only thread linker will be missing the user's own sent messages. Our fix is a **JWZ-style heuristic**: subject normalisation + participant overlap + 90-day window. Works in practice but is technically fragile to subject rewrites. Document this prominently — future agents will rediscover it.

2. **Liquid Glass: never custom shadow/border on a glass surface.** System renders edge effects. Multiple toast / popover designs got "fixed" before this rule landed in CLAUDE.md. Two of four review agents converged on this independently.

3. **LSUIElement apps need a manually-installed main NSMenu** for ⌘A/C/V/X/Z/B/I/U to work in any TextField. Without it, Cocoa's standard text-editing key equivalents don't dispatch. `AppDelegate.installMinimalMenuBar()` is load-bearing — do not remove.

4. **WKWebView's internal NSScrollView consumes scroll events even when content can't scroll** (elastic bounce). Three-layer fix: PassthroughWebView subclass + recursive `verticalScrollElasticity = .none` walk + CSS `overflow: hidden`.

5. **`currentAccount = nil` is the unified-inbox sentinel.** Compose's `fromOverride` is per-compose, separate from inbox view. Reply / Forward set fromOverride to `accounts.first { $0.email == msg.account_email }` so identity follows the message.

6. **Tests are stale.** `Tests/MinimailTests/Fixtures.swift` predates v0.7.0 schema additions. `swift test` will fail. Either update fixtures or mark xfail before adding new tests.

7. **Two old Minimail processes were leaking** at one point (PIDs from earlier sessions). If app misbehaves: `pkill -x Minimail && open -a /Applications/Minimail.app`.

8. **Codex sessions tend to spend their context on file exploration** rather than producing clean synthesis blocks. Ask for the report up-front + bound the exploration ("read these files, then report"). Multiple Codex runs this session ended in mid-file-listing.

### Boris-overrides on agent recommendations

The product-gap agent listed **read receipts / email tracking** as MUST-HAVE. **Override: do not ship.** Reasoning:

### Architecture risks (my analysis — Codex didn't synthesize)

1. **AppState.swift god-object — 1214 lines.** Bundles: substate ownership, CLI orchestration, compose lifecycle (autosave + fromOverride + undo-send pipeline), draft management, contact index, transient toasts, account quick-switcher, forward-attachment download, reply quote builder, forward body builder, HTML stripper, UserDefaults persistence, logger declarations. **Decompose into:** `MailService` (CLI wrappers), `ContactIndex` (autocomplete), `UndoSendQueue` (pending send + toast), `ComposeService` (quote/forward builders), `PreferencesStore` (UserDefaults). AppState becomes a thin coordinator. **Effort: 1–2 sessions. Should-have not must-have for v1.0.**

2. **InboxView.swift mixed concerns — 810 lines.** Extract: `DayGrouping`, `DateFormat`, `IconButtonStyle`, `AccountAvatar`, `FolderTab`, `MessageRow` each to own file. InboxView becomes ~250 lines. **Effort: 1 session. Nice-to-have.**

3. **Race condition risk:** poll timer + user-initiated refresh can run `refreshInbox` concurrently. Should serialize via a single Task handle (cancel previous, replace). **Effort: 30 min.**

4. **HTMLBodyView resource lifetime:** cached static rule list survives app lifetime (correct), Coordinator holds weak webView ref (correct), but the `rulesPending` array of closures could in theory leak if the WKWebView dealloc happens between compile-completion and flush. Add a defensive nil check after `await`. **Effort: 15 min.**

5. **Rust `inbox.rs` thread heuristic — 1034-line file, the most subtle code in the repo, no tests.** Extract `inbox_thread()` orchestrator into smaller pure functions (subject normaliser, participant matcher, time-window filter) so each can be unit-tested. **Effort: 1 session including tests. HIGH PRIORITY before v1.0.**
