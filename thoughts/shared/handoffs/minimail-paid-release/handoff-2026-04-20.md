## Key Decisions Made (not obvious from code)

1. **No Package.swift for Loop ‚Äî it's Xcode project based.** Luminare (MrKai77/Luminare) is a separate SPM package. We're NOT adopting Luminare; we use Apple's native `.glassEffect()` directly with `macOS 26` as min target.

2. **Deployment target locked to macOS 26.0.** Do NOT add `#available(macOS 26, *)` guards ‚Äî fallback branches are dead code. Four review agents converged on this.

3. **No custom shadows / borders on glass surfaces.** Apple HIG rule. Violated this twice in early toast + popover designs; fixed.

4. **LSUIElement apps break text shortcuts.** The non-obvious mistake everywhere: `NSApp.mainMenu == nil` means NSTextField never receives ‚åòA / ‚åòC / ‚åòV / ‚åòZ. Solved by `AppDelegate.installMinimalMenuBar()`. Don't remove that function.

5. **Bare-letter keyboard shortcuts at RootView scope are poison** ‚Äî they swallow typed text in every descendant TextField. Only OK in leaf views with no text descendants (e.g. `s` in reader). Global help is `‚åò/`, not bare `?`.

6. **CLI architecture is "app is a shell around the CLI"** ‚Äî mirrored by McClaw. Don't stuff business logic into Swift; keep it in email-cli. Swift only handles UI + process management.

7. **AsyncDataCollector (pipe drain) is load-bearing.** Don't "simplify" the Process wrapper in `EmailCLI.swift:runPlainData` ‚Äî pipe buffers are 64KB, CLI emits larger payloads, deadlocks are silent and intermittent.

8. **NSAttributedString HTML export is acceptable wire format.** Resend accepts ~1KB Cocoa HTML fine. Don't overthink it.

9. **Cc/Bcc hidden by default** (per Apple Mail / iOS Mail convention). Auto-expands when a draft has Cc/Bcc content already.

10. **`pendingSend: PendingSend?` on AppState** is the undo-send queue. Deadline is `Date + 10s`. Task fires `firePendingSend()` after sleep; `undoPendingSend()` cancels and restores compose state verbatim.

11. **Reviewed the open-source SwiftUI ecosystem for design inspiration** ‚Äî user concluded there's little to learn externally. Apple's built-in apps (Messages, Mail, Notes, Safari) on macOS 26 are the real reference. Only external codebase worth reading is **MrKai77/Luminare** (translucent design system, 156‚òÖ, SPM-installable) ‚Äî but we don't *need* it for Minimail.

- **Don't add `#available(macOS 26, *)` guards** ‚Äî deployment target is 26.0, guards are dead weight. Four review agents converged on this. The fallback branches inside them will also never execute.

- **Don't bind bare letters with `modifiers: []` in RootView** ‚Äî swallows typed punctuation in every text field. `‚åò/` for help (not bare `?`).
